Project Description and Overview

The Wayback Public Library System is a comprehensive library management application designed to handle book inventory, patron management, and borrowing activities. Developed as part of a collaborative project, it utilizes modern web technologies and adheres to agile practices for efficient delivery.

My Role: I served as the team leader and played a pivotal role in both front-end and back-end development. I guided the team through the project using Scrum methodology, facilitated the use of GitHub and IDEs, and contributed significantly to the development and implementation of key features. My responsibilities also included ensuring effective communication within the team and overseeing the integration of various components to achieve the project goals.

Features

Book Management: The system allows for the addition, updating, and removal of books. The application checks for duplicate entries and validates book details to prevent incorrect data from being saved.

Patron Management: It includes functionalities to add and update patron information. The system ensures that email addresses and phone numbers follow a valid format to maintain data integrity.

Borrowing and Returning Books: Patrons can borrow and return books, with the system enforcing rules such as no overdue books and checking for existing borrowed books.

Search and Filter Options: Users can search and filter books based on various criteria, including title, author, and genre. The search functionality is optimized for quick and accurate results.

Administrative Controls: Admins can manage users, oversee borrowing activities, and generate reports on system usage. Permissions and roles are defined to ensure appropriate access levels.
